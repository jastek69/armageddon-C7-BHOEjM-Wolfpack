## Where to find logs when the environment is up

- **CloudTrail** — infra changes (TGW, RDS, routes, security groups)
- **Application** — Flask logs on EC2 via systemd: `journalctl -u flask.service` or `/var/log/flask.log` if configured with the CloudWatch agent
- **ALB** — access logs to S3 if enabled in the ALB attributes
- **RDS** — CloudWatch metrics and optionally Performance Insights

For this lab the primary evidence is `audit-pack/04_cloudtrail-change-proof.txt` and the CLI outputs in the other audit-pack files.
